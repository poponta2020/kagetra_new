## 1. 対象と狙い
- **対象画面:** [/players/[id]](../../../apps/web/src/app/(app)/players/[id]/page.tsx)（tournament-results Task5 の読み取り専用MVP）
- **現状の不満:** ①キャリアの集計・サマリーが無い ②出場大会が多い選手だと試合表が延々続いて一覧が長すぎる
- **狙い:** パッと全体像（集計）が掴め、長い履歴もスクロール最小で流し見できる
- **主ユーザー / 主な使い方:** 他選手の履歴を眺める

## 2. 採用した方向性
- **A（エディトリアル／年タイムライン）** をユーザーが Claude Design 上で改良して確定。
- **不採用 B（大会カード）:** 箱が積み上がり「眺める」には情報密度が低い。**C（ダッシュボード）:** 集計に振りすぎて一試合ずつ追う用途と乖離。
- 採用案の現物＝上記モック。本仕様はそのモックを実装基準として言語化したもの。

## 3. レイアウト構成（上→下）
1. **戻る導線** — 「← 選手検索へ戻る」テキストリンク。
2. **サマリー（箱なし・和紙地に直接）**
   - 氏名（serif 23px）＋右に**現在の級**（最新出場の級。例「（A級）」、薄色）／下に所属会（最新の所属、例「札幌かるた会」）。
   - `通算 182勝96敗`（serif・勝=藍/敗=朱）と `勝率 65.5%`（serif・藍）をベースライン揃えで横並び。
   - 最下に薄い1行：`47大会 ・ 優勝5 ・ 入賞18 ・ 2010–2026（16年）`（en ダッシュ）。
   - 区切りは 1px ヘアライン（`#E6DDC4`）。チップ・カード枠は使わない。
3. **年セクション（暦年・新しい年から）**
   - 見出し＝**フラット下線＋sticky**（スクロール追従、`position:sticky; top:0`）。開いている年は見出しを藍に。`▾/▸` トグル。右に `N大会 ・ W勝L敗` のミニ集計。
   - **展開単位＝年**：年見出しタップでその年を開閉。開くと**その年の全大会の試合表をまとめて表示**（大会ごとの個別トグルは持たない）。
   - 既定：**最新年のみ展開**、以降は折りたたみ。末尾に「…さらに <最古年>年まで」。
4. **大会ブロック（展開時）**
   - 1行ヘッダ：`M/D ＋ 大会名（「第N回」除去・級letterを後置, 例「北海道選手権A」）`、右端に**順位**（優勝/準優勝＝藍ボールド・ベスト_＝薄色）。
   - 直下に試合表。大会間は 1px ヘアライン。
5. **試合表（フラット・枠線なし）**
   - 並び＝**決勝→1回戦の降順**（直近の試合が上）。
   - 列：回戦ラベル（薄色）／相手名＋**所属会**（薄い括弧, 例「渡辺 大輔（東京暁星会）」）／枚数＋勝敗を**1トークン**＝`○12`（勝=藍）/`×7`（負=朱）。
   - **相手名タップ遷移（R1）:** `opponent_participant_id` 解決済みの相手名はタップで その選手の `/players/[id]` へ。見た目は**黒（ink）・下線なし＝通常テキストのまま**（明示 affordance なし）。未解決相手も同一見た目（遷移しない）。所属会の muted 併記は不変。タップ範囲＝氏名文字列。

## 4. 使用コンポーネント
- **方針転換あり:** 採用案は Card/Pill/GradePill を**使わず**、和紙地に直接組むフラット表現。実装は本ページ専用のマークアップ＋スタイルになる（既存プリミティブの流用は最小）。
- **新規（本ページ内 or 小コンポーネント化を検討）:**
  - sticky 年ヘッダ（開閉状態を持つ）
  - 年グループ（展開＝当年の全大会＋試合表）
  - フラット試合表（`○N/×N` トークン・相手＋所属会）
  - サマリーブロック（導出値の集計表示）
- 既存と被るもの（順位・級の見せ方）が**Pill から text へ**変わる点は意図的（design.md のピル前提から逸脱）。レビュー時に共通化要否を判断。

## 5. 状態（state）
- **通常:** 上記。
- **出場記録なし:** サマリーは通算0/0・大会0 等の最小表示、年セクションは出さず「出場記録がありません。」。
- **長大（多数年・多数大会）:** 年で畳むことで初期スクロールを最小化。最新年のみ展開。`overflow` はページスクロール（モバイル）。
- **開催日不明（eventDate null）:** 年に割り当てられない大会は末尾に**「開催日不明」グループ**としてまとめる（年降順の最後）。
- **順位が導出不能な級:** §6 のフォールバック（保存 final_rank）。両方無ければ順位は非表示（大会名・W-L は出す）。
- **不戦勝/棄権（status=walkover/forfeit, score_diff null）:** スコアトークンを `不戦勝`/`棄権` に分岐。勝敗集計には**含めない**（既存仕様踏襲）。

## 6. 必要データと導出
### 表示フィールド
| 項目 | 出所 |
|---|---|
| 氏名 | `players.display_name` |
| 現在の級 / 所属会 | 最新出場（eventDate 最大）の `tournament_classes.grade` / `tournament_participants.affiliation` |
| 通算勝敗・勝率 | `matches` の `status='normal'` を勝(win)/敗(lose)集計（既存 `getPlayerRecord` 済） |
| 出場大会数 | その選手の `tournament_participants` 件数（同一大会で複数級は稀。当面は参加数=大会数の近似で可、要レビュー） |
| 活動年スパン | 参加の `tournaments.event_date` の最小〜最大の年（null は除外） |
| 大会名/級/開催日 | `tournaments.name`（「第N回」除去）/ `tournament_classes.grade` / `tournaments.event_date` |
| 大会ごとの W-L | その participation 内 `matches`（status=normal）から集計 |
| 相手名＋所属会 | `matches.opponent_name` ＋ `matches.opponent_participant_id`→`tournament_participants.affiliation`（解決時のみ。未解決は所属空欄） |
| スコアトークン | `result`＋`|score_diff|`（status 分岐） |

> **クエリ拡張:** 現行 `getPlayerRecord`（[queries.ts](../../../apps/web/src/lib/players/queries.ts)）は `opponentName` のみ取得。**opponent participant の affiliation を join** する拡張が要る。

### 順位（優勝/入賞/ベストN）の導出 ★本リデザインの追加機能
**自由記述の final_rank に依存せず、対戦結果から導出**するのが第一方針。
- **対象＝単純トーナメント（シングルイリミ）の級。** 級内 `matches` から `maxRound`（=決勝）を求め、各選手の**最終試合の round と result**で判定：
  - 決勝に勝ち → **優勝**
  - 決勝で負け → **準優勝**
  - 準決勝で負け → **ベスト4**／準々決勝で負け → **ベスト8**／round (maxRound−k) で負け → ベスト(2^(k+1))
  - `round_label` が意味ラベル（決勝/準決勝/準々決勝）なら優先採用、無ければ round 数と maxRound から算出。
- **入賞の定義＝ベスト8以上**（優勝・準優勝・ベスト4・ベスト8を入賞として数える）。サマリーの「入賞N」「優勝N」はこの導出結果の集計。
- **フォールバック:** リーグ戦・順位戦・予選+本戦の混在・3位決定戦・シード/不戦・データ欠け 等で**導出が一意に決まらない級は、保存済み `final_rank`（自由記述）をそのまま表示**に使う。導出も final_rank も無ければ順位非表示。
  - 導出可否の判定はヒューリスティック（例：各非優勝者の敗北が1回で停止しブラケット状か／`round_label` に「リーグ/順位/予選」を含まないか／決勝の勝者が一意か）。閾値はテストで詰める。
- **`final_rank` 列は当面残す**（フォールバック用）。**列の物理削除はやらない**＝全フォーマット監査を前提とする別タスクに分離（スコープ混ぜない）。

## 7. インタラクション / レスポンシブ
- 年見出しタップで開閉（sticky 追従）。タップ領域は見出し行全体。
- 375px 基準。横スクロール禁止：大会名は省略記号で切る、試合表は固定3列ではみ出させない。
- アニメは 150ms 以内（背景/影のみ）。

## 8. ガードレール準拠メモ
- washi×藍墨・純白/真黒なし・絵文字なし・です/ます・代名詞回避 — 準拠。
- serif＝氏名・大きい数字（通算/勝率）、sans＝その他 — 準拠。
- 二値セマンティック：○=藍(success)/×=朱(danger)。順位の正の強調は藍。
- **意図的な逸脱:** 年スパンを波ダッシュ `〜` ではなく **en ダッシュ `–`**（ユーザー承認済み）。順位/級を Pill ではなく text 表現にした（エディトリアル方針）。

## 9. 残課題・実装への申し送り
- **順位導出のエッジ:** 3位決定戦・シード/不戦・リーグ/順位戦/予選+本戦の検出とフォールバック切替。テスト必須（既知フォーマットの実データで導出 vs 保存値を突合）。
- **相手所属の join:** `getPlayerRecord` 拡張（opponent participant → affiliation）。
- **出場大会数の数え方:** 同一大会・複数級参加の重複をどう扱うか（当面 参加数=大会数 近似）。
- **既定展開年ポリシー:** 最新年のみ展開で確定（要レビュー）。
- **パフォーマンス:** 多作な選手は matches が多い。級ごとの順位導出を N+1 にせず一括取得＋メモリ集計で。
- **共通化判断:** フラット表現が他画面（大会別結果 等）でも要るなら小コンポーネント化を検討（このリデザイン内では本ページ専用で可）。
- **final_rank 列削除は別タスク**（監査前提）。本リデザインに混ぜない。
