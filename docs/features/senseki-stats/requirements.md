# senseki-stats 要件定義書

BottomNav「戦績」を「統計」に改称し、その配下を **4 セクション**（【閲覧】選手検索・
大会結果／【統計】選手ランキング・大会統計）に再構成する。収録済みの全国大会データ
（1,496 大会 / 819,703 対戦 / 47,709 選手 / 367,675 参加）を、個別に引く（閲覧）／
横断で集計する（統計）の2軸で見られるようにする。

## 1. 概要

- **目的:** 戦績タブ（現状 `/players` の選手検索1画面のみ）を「統計」タブに改称し、
  4 セクション構成にして、大会結果の閲覧と横断統計を提供する。
- **背景:** tournament-results / bulk-result-import で大量の全国大会データが入ったが、
  個人の戦績詳細（`/players/[id]`）以外に閲覧・集計の場がない。
- **対象データ:** 全国データ（全選手・全大会）。会員フィルタは無し（§6）。

## 2. ユーザーストーリー

- **対象ユーザー:** 全ログインユーザー（会員含む）。戦績タブの audience を継承。
- **やりたいこと（2軸）:**
  - 【閲覧】特定の選手 or 特定の大会を引いて、その全結果を見る。
  - 【統計】選手ランキングや年別推移など、横断的な集計を期間・級で絞って眺める。
- **ゴール:** 大会結果・ランキング・歴代結果のどこからでも、各選手の戦績詳細へ
  自然に遷移できる。

## 3. 機能要件

### 3.1 ナビゲーション（ナビ地図）
- **BottomNav「戦績」→「統計」に文言変更。**
- 「統計」配下を **4 セクション**に分割。概念上 2 グループ：
  - **【閲覧】** ① 選手検索 ／ ② 大会結果
  - **【統計】** ③ 選手ランキング ／ ④ 大会統計
- **ルート:**
  | セクション | ルート | 備考 |
  |---|---|---|
  | ① 選手検索（着地点） | `/players` | 既存据え置き |
  | 戦績詳細 | `/players/[id]` | 既存据え置き（プッシュ表示） |
  | ③ 選手ランキング | `/players/ranking` | 新規 |
  | ② 大会結果（年別ビュー） | `/tournaments` | 新規・大会別とトグル |
  | 大会別ビュー（シリーズ一覧） | `/tournaments/series` | 新規・年別とトグル |
  | 大会詳細（入賞者＋級タブ） | `/tournaments/[id]` | 新規（プッシュ表示） |
  | シリーズ詳細 | `/tournaments/series/[id]` | 新規（プッシュ表示） |
  | ④ 大会統計（メイン全体サマリー） | `/tournaments/stats` | 新規 |
  | 大会統計 図詳細（級別比較） | `/tournaments/stats/<score/competitors/participations>` | 新規（プッシュ表示） |
- 静的セグメント（ranking / series / stats）は動的 `[id]` より優先されるため
  `/players/[id]` `/tournaments/[id]` と衝突しない。
- BottomNav「統計」は href=`/players`、active 判定は `/players` と `/tournaments`
  の両プレフィックス。
- セクション切替コントロール（segmented 等）は 4 セクションの**トップ**にのみ表示。
  戦績詳細・大会詳細・シリーズ詳細は**コントロール無しの全画面プッシュ**（戻る導線つき）。
  **コントロールの形・グルーピング表現（4 フラット or 閲覧/統計の 2×2）は design-spec 管轄。**

### 3.2 横断フィルタ
- **期間（年 from–to）:** ③選手ランキング・④大会統計（メイン/詳細とも）に適用。
  `tournaments.event_date` の年。event_date 無し大会は期間集計から除外。
- **級（A–E、複数選択可）:** **③選手ランキングのみ**に適用。
- **④大会統計は級フィルタを使わない:** メインは全級固定、詳細は全級＋各級を並置して比較する
  （級は絞り込みではなく比較軸）。
- 対象は全国データ固定。①選手検索・②大会結果（個別閲覧）・シリーズ詳細にはフィルタ非適用。

### 3.3 ① 選手検索【閲覧・既存流用】
- 既存 `searchPlayers` / 検索フォーム / 結果一覧 / `/players/[id]` 戦績詳細をそのまま
  流用。本機能ではセクションコントロール配下に置くのみ（検索ロジック変更なし）。

### 3.4 ② 大会結果【閲覧・新規】

#### 大会一覧（`/tournaments`）— 年別 ／ 大会別 の2ビュー切替
- **年別ビュー（`/tournaments`・既定）:** 全 1,496 大会を開催日降順・**年セクション**で一覧。
  各行＝大会名／開催日・会場／**級構成（A〜E のトーンドット）**＋参加者数。中止回は「中止」表示・
  参加者数なし。行タップ → 大会詳細。
- **大会別ビュー（`/tournaments/series`）:** 同じ大会（シリーズ）を1行に束ねた 180 系列一覧。
  各行＝系列名／累計開催回数・回次範囲・直近開催年・状態内訳（開催/中止/未確定）。行タップ →
  シリーズ詳細。**年別↔大会別はトップのトグルで切替**（大会名検索は共通）。
- **シリーズ詳細（`/tournaments/series/[id]`）:** 回次一覧（第N回・開催年・状態・あれば優勝者）＋
  参加者数推移。結果データのある回 → 大会詳細へリンク（無い回は「データなし」、中止回は中止表示）。

#### 大会詳細（`/tournaments/[id]`）— タブ＝入賞者 ＋ 級ごと
旧サイトの「入賞者タブ＋級タブ」構造を踏襲（見た目はアプリ意匠へ刷新）。**タブ＝入賞者 ｜ 級（A/B/…）**
（横スクロール）。**同一級が分割運営される場合はタブを A1／A2… に分ける**（edition の級ブロック単位）。
- **入賞者タブ（既定）:** 級（ブロック）ごとに 優勝→2位→**3位（同着複数）**→**4位（同着複数）** を
  導出ブラケット（優勝=bracket1／2位=2／3位=ベスト4／4位=ベスト8）で表示。導出不能級は `final_rank`
  フォールバック。所属併記・行タップ → 戦績詳細。
- **級タブ（A/B/… or A1/A2）:** その級の**全結果**を、旧サイトの対戦表（選手×回戦のクロス表）を
  **踏襲**して表示。**行＝選手（勝ち上がり順）／列＝回戦**、セル＝○/× ・ 相手 ・ **枚数差**
  （不戦は相手/枚数なし、status=normal）。**氏名列を固定（sticky）し回戦は横スクロール**、敗退後は
  空欄＝勝ち残りが**逆三角形**に見える（旧UIの一目で分かる特性をユーザーが重視）。決勝進出者は全経路
  を実データ、途中敗退者は**敗退対戦（実データ）＋そこまでの勝ち上がり（○のみ・相手表示は省略）**で
  到達回戦を正確表現（相手不明の対戦は捏造しない）。○＝藍／×＝中立インク。選手タップ → 戦績詳細。
  （初期案の「選手ごとの経路チップ」はユーザー指摘で破棄→クロス表復元。）

### 3.5 ③ 選手ランキング【統計・新規】
- 指標切替（行タップ → 戦績詳細）：
  - **出場回数** ／ **勝利数**（normal の win）／ **勝率**（最低 20 試合で足切り）／
    **対戦数**（normal の試合数）／ **優勝回数**（bracket=1）／ **入賞回数**（bracket≤8）
- 期間・級フィルタ連動。表示は **TOP 100 ＋「もっと見る」**。同値は同順位扱い、
  並びは値降順 → 表示名。

### 3.6 ④ 大会統計【統計・新規】
**メイン全体サマリー → 一部の図は詳細（全級＋各級の比較）へドリル**する構成。
用語：**競技人口**＝ユニーク選手数 ／ **大会参加人数**＝延べ参加。**大会統計は級フィルタを使わない**
（絞り込みではなく、級は図内 or 詳細での比較軸）。**期間（年）フィルタはメイン・詳細とも有効。**

#### メイン全体サマリー（`/tournaments/stats`）— 全級・期間フィルタのみ
- **絶対数カード（4）:** 競技人口 ／ 収録大会数 ／ 総対戦数 ／ 大会参加人数。
- **図（6つ）。2 群に分かれる:**
  - **【この画面で完結＝詳細なし】**（級が図内で表現済み or サマリー専用）:
    1. **級別構成の推移** … x=年、A〜E の100%積み上げ（藍→砂トーンランプ）。図内で級別比較が完成。
    2. **新規参入者の推移** … x=初出場年、y=新規選手数 の棒。**この画面のみ**（級別詳細は持たない）。
       初出場年＝その選手が初めて大会記録に現れた年。**収録開始の 2010 年は既存選手を含むため
       集計から除外（2011〜）**＝UI に注記。
    3. **一人当たり 平均年参加数（級別）** … **x=級（A〜E）**、y=大会/年（その人がその年に出た大会数の
       平均）。集計対象期間はメインの期間フィルタに連動。x が級なので図内で級別比較が完成＝詳細なし。
  - **【全級の集計＝級別比較は詳細へドリル】**:
    4. **スコア統計** … x=枚数差（1〜25・1枚刻み=25本）、y=試合数 のヒストグラム（平均線つき）。
    5. **年別 競技人口推移** … x=年、y=競技人口 の棒。
    6. **年別 大会参加人数推移** … x=年、y=大会参加人数 の棒。
  - 図 4〜6 のみ右上「級別比較 ›」で詳細へドリル可能。

#### 図詳細：全級＋各級の比較（`/tournaments/stats/<metric>`・metric=score/competitors/participations）
- 図 4〜6 それぞれの詳細。**全級＋A〜E を並べて比較**（スモールマルチプル＝各系列のミニ図を縦に並べる）。
  score=枚数差ヒスト、competitors/participations=年推移。
- **期間（年）フィルタ可・級フィルタなし**（全級＋各級を常に並置）。

## 4. 技術設計

### 4.1 順位の事前計算（共通基盤・最初に作る）
- **方式:** `tournament_participants` に `derived_bracket smallint`（1=優勝/2=準優勝/
  4/8/16…、導出不能級は null）を追加。
- **理由:** ②大会詳細の級別順位・③優勝/入賞ランキング・②シリーズ歴代優勝者を**全部**
  支える。とくに③のランキングに期間・級フィルタを効かせるには参加グレインで bracket を
  保持し、クエリ時に WHERE で絞って集計する必要がある（選手単位の事前集計表ではフィルタ
  不能）。②大会詳細の順位表示も保存値の読み出しで軽くなる。
- **計算:** 取込承認（materialize）時に、級ごとに既存 `lib/players/placement` の
  `isDerivableClass` / `derivePlacement` を回し、各 participant に bracket を書く
  （戦績詳細と同一ロジックを流用＝順位定義の単一ソース）。
- **バックフィル:** 既存 367,675 参加を一回限りのスクリプトで埋める（全級を走査して
  derivePlacement → UPDATE）。本番反映は dump/apply 方針（[[project_bulk_load_handover]]）。

### 4.2 データ層 / API（読み取り専用・サーバー集計）
新規クエリ群（`apps/web/src/lib/stats/*` 仮）。既存 `searchPlayers` / `getPlayerRecord`
は流用。フィルタ型 `{ yearFrom?, yearTo?, grades?: ('A'|..'E')[] }` を共通化。
- `getStatsOverview(filter)` … メイン全体サマリー：絶対数4種＋6図（級別構成推移／新規参入者推移[初出場年・
  2010 は左側打ち切り]／一人当たり平均年参加数[x=級 A〜E]／スコアヒスト／年別競技人口／年別大会参加人数）。
  期間フィルタのみ。新規参入者と級別一人当たりは詳細を持たない
- `getStatsDetail(metric, filter)` … 図詳細：metric=score/competitors/participations。
  **全級＋各級（A〜E）**の系列を返す（score=枚数差ヒスト、他=年推移）。期間フィルタのみ
- `getPlayerRanking(metric, filter, limit, offset)` … 指標別ランキング（優勝/入賞は
  `derived_bracket` 集計）
- `getTournamentList(query?, year?)` … 大会一覧（年別ビュー）
- `getTournamentResults(tournamentId)` … 級ブロック（A/B/…、分割時 A1/A2）ごとに
  {入賞者=順位(導出bracket・優勝/2位/3位/4位), 対戦表=選手×回戦のクロス表(勝ち上がり順・各セル○×・相手・枚数差)}。
  入賞者タブ＝bracket 集約、級タブ＝クロス表（氏名固定列＋回戦横スクロール）
- `getSeriesList()` … 系列 → {累計開催, 回次範囲, 直近年, 状態内訳}（大会別ビュー）
- `getSeriesDetail(seriesId)` … 回次一覧（年/状態/優勝者/大会リンク）＋ 参加者数推移

### 4.3 DB
- migration: `tournament_participants.derived_bracket smallint`（null 許容）追加。
  ランキング集計（player 別 count・年/級 join）を支える index を付与。
- backfill スクリプト（test/dev は push、本番は migrate ＋ dump/apply）。

## 5. 影響範囲

- **schema 追加:** `tournament_participants.derived_bracket`。materialize（取込承認）に
  書き込み処理を追加。戦績詳細（`getPlayerRecord`）は保存値を使うよう将来最適化可能
  （必須ではない・現行 JS 導出のままでも整合）。
- **ナビ:** `bottom-nav.tsx` の文言（戦績→統計）＋ active 判定に `/tournaments` 追加。
- **新規ルート:** `/players/ranking`・`/tournaments`・`/tournaments/[id]`・
  `/tournaments/series`・`/tournaments/series/[id]`・`/tournaments/stats`。
- 既存 `/players`（検索）・`/players/[id]`（詳細）は据え置き（流用）。

## 6. 設計判断の根拠

- **対象=全国データ:** `players.user_id` は v1 で基本 null（会員↔選手紐付け未実装）。
  会員のみ統計は別途紐付け機能が前提でスコープ大のため v1 は全国対象。
- **閲覧/統計の2軸 × 4 セクション:** 「各大会の全結果を見る」は集計ではなく閲覧、という
  ユーザーの整理に従い、選手検索・大会結果（閲覧）と 選手ランキング・大会統計（統計）に
  分離。
- **タブ名は「統計」:** 閲覧も含むがユーザー指定で確定。
- **順位は参加グレインで事前計算（derived_bracket）:** 期間・級フィルタとの両立 ＋
  大会詳細/歴代優勝者の共通基盤化のため（§4.1）。順位定義は戦績詳細と同一ロジック流用。
- **URL基底は `/players`・`/tournaments` 据え置き/新設:** `/players/[id]` 詳細
  （senseki-detail-redesign の `?from=` 遷移）を壊さないため移設しない。
- **視覚設計は /design-screen に委譲:** カード/表/グラフ/一覧主体で視覚比重が高い。

## 7. スコープ外（v1 では作らない）

- 会員↔選手の紐付け、会員フィルタ。
- 段位分布・連覇記録・最多対戦カード・所属会別/都道府県別ランキング・直近大会フィード・
  殿堂（過去ブレストの未採用案）。
- 大会詳細のトーナメント表（ブラケット）ビジュアル（順位一覧＋全対戦リストで代替）。

## デザインへの宿題（→ /design-screen senseki-stats）

**すべて解決済み（design-spec.md `status: locked` に反映）。収束ゲート＝空。** 主な確定：
- 4 セクション＝**フラット4タブ（下線）**シェル（案A）。
- 選手ランキング＝横スクロール指標チップ＋1行フィルタ＋順位リスト（TOP100＋もっと見る）。
- 大会統計＝**メイン全体サマリー（4カード＋6図）＋図詳細（級別比較スモールマルチプル）**。
- 大会一覧＝**年別／大会別トグル**。大会詳細＝**入賞者タブ＋級タブ（クロス表）**。シリーズ詳細＝回次一覧＋参加者数推移。
- 期間フィルタ＝1行（シート想定）。級色＝藍→砂トーンランプ。朱はデータ装飾に使わない（中止/締切等のみ）。
