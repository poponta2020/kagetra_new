# 招待URL会員登録 デザイン仕様（design-spec）

> `/design-screen` 出力の確定仕様。**ロジック/データ/検証/DB/API の正典は [requirements.md](./requirements.md)**（本書では再記述しない）。本書は視覚（レイアウト・状態の見え方）を定義。

## 1. 対象と狙い
- **対象画面:** `/register/[token]`（招待URLからの会員セルフ登録。(app) シェル外＝AppBar/モバイルナビ無し）。
- **狙い:** 和紙×藍墨ブランド（A-flat・`design.md §2.5 アンチスロップ規約`）に整え、招待された新規会員が LINE認証→プロフィール入力（氏名・級・段位・全日協登録情報）を一度で完了。

## 2. 採用した方向性
- **A案「フラット（脱カード）」** — 確定モック `preview/invite-register-a.html`（Claude Design `招待登録 (Redesign)` グループ）。
- フォームを囲むカード箱・影を撤去し、**ヘッダ罫線＋余白＋タイポ**で階層化。入力は**アンダーライン式**、級・段位・性別は**下線セグメント**、全日協は**箱なしチェック**、「LINE認証済み」は**テキスト＋CSSチェック**。和紙 surface `#F4EFE3`・藍墨・serif ワードマーク「かげとら」は維持。角丸控えめ（ボタン radius 4）・影なし。
- **不採用:** B案（ウェルカム型ステッパー）/ C案（情報リッチ）＝装飾過多・文字過多で `§2.5` と不整合。
- **before スナップショット:** `preview/invite-register-current.html`。

## 3. レイアウト構成（②フォーム・上→下）
箱なしブロック（`max-width:340px`・上揃え `padding-top:48px`）。
1. **ヘッダ:** serif ワードマーク「かげとら」＋サブ「**北大かるた会 大会管理アプリ**」、下罫線。
2. **お名前:** **2×2 アンダーライン入力** — 姓（漢字）/ 名（漢字）/ せい（ひらがな）/ めい（ひらがな）。
3. **級:** 下線セグメント A/B/C/D/E。補助「後から会員ページでいつでも変更できます。」
4. **全日協 登録チェック**（級＝A/B/C のときのみ表示・既定 ON）: 箱なしチェック「全日本かるた協会（全日協）に登録済み」。
5. **段位**（級＝A のときのみ表示）: 下線セグメント 四/五/六/七/八段（既定 四段）。補助「A級の方のみ。該当する段位を選択してください。」
6. **全日協登録情報**（全日協チェック ON のときのみ・罫線見出し「全日協登録情報」）:
   - **性別**: 下線セグメント 男性/女性（2択のみ）。
   - **生年月日**: date 入力。
   - **電話番号**: tel 入力。
   - **郵便番号** ＋「住所を検索」ボタン（右）: 入力後に**住所1**を自動補完。検索メッセージ行。
   - **住所1**（丁目・番地まで）/ **住所2**（マンション名・部屋番号）の2分割。住所2 ラベル横に小チェック「集合住宅ではない（一軒家）のため未入力」→ ON で住所2 を無効化・クリア。補助「免状・かるた展望などの郵送先になります。」
7. **登録する**（藍 `#2B4E8C`・全幅・radius 4）。

**①未ログイン:** ヘッダ ＋ リード「招待リンクから会員登録します。まず LINE アカウントで認証してください。」＋ LINE緑ボタン ＋ フッタ「認証のあと、お名前と級を入力します。」
**③リンク無効（共通）:** `invite-register-error.html`。導線なし＋朱枠でエラー＋管理者依頼。

## 4. 使用コンポーネント
- **撤去:** 登録カード（白/和紙箱＋影）。Shell の `bg-white rounded-lg shadow-lg` をやめフラット化。
- **新規フラット primitive:** アンダーライン TextField／下線セグメント（級・段位・性別共用）／箱なしチェックボックス（藍 CSS チェック）／郵便番号＋検索ボタン行（`.zip-row`/`.zip-btn`）。
- **流用:** serif ワードマーク・和紙背景・ブランド/LINE ボタン色。
- date/tel/numeric は適切な inputmode を付与。

## 5. 状態（state・級と全日協チェックで段階表示）
- **級 D/E:** 氏名＋級のみ（全日協チェック・段位・PII は非表示）。
- **級 B/C:** ＋全日協チェック（ON で PII ブロック表示）。段位は非表示。
- **級 A:** ＋段位（必須・既定四段）＋全日協チェック（ON で PII）。
- **全日協チェック OFF:** PII ブロックを隠し未収集（`zenNichikyo=false`）。
- **PII（チェック ON 時）:** 性別/生年月日/電話/郵便/住所をすべて表示（要件で全必須）。
- **住所検索:** ボタン押下→検索中→成功（住所反映・番地追記を促す）／失敗（手入力を促す）メッセージ。
- **送信中/エラー/③無効:** 現状踏襲（disabled・朱系エラー・無効リンクは導線なし）。

## 6. 必要データ（正典＝requirements.md §4.2）
視覚に必要な対応のみ:
- 既存 `users`: `name`(UNIQUE/合成表示名)・`grade`・`gender`(男女)・`dan`・`zen_nichikyo`。
- 新規列: `family_name/given_name/family_kana/given_kana`・`birth_date`・`phone`・`postal_code`・`address1`・`address2`。
- 郵便番号→住所: サーバー経由 zipcloud（requirements §4.1）。

## 7. インタラクション / レスポンシブ
- **段階表示ロジック:** 級選択で段位（A のみ）・全日協チェック（A/B/C）を出し分け。全日協チェックで PII ブロックを開閉。級を下げると上位限定項目は隠し値リセット（段位・PII）。詳細な保存条件は requirements。
- **住所検索:** 郵便番号7桁→サーバー経由 zipcloud で住所補完、失敗時は手入力。150ms 程度の素直な遷移。
- **モバイル 375px:** 氏名 2×2＝`1fr 1fr`。郵便番号行は入力＋ボタン横並び。タップ領域確保。`maximum-scale=1` 済（[[impl_disable_pinch_zoom]]）。

## 8. ガードレール準拠メモ
- 配色: 和紙 surface・藍墨。朱は必須/エラーのみ。LINE 緑は外部ブランド踏襲。藍チェック。
- フォント: serif=ワードマーク。本文/入力は sans。
- フラット: `design.md §2.5`（箱・影・装飾を排し余白/罫線/タイポ）。絵文字なし（チェックは CSS 描画）。
- 日付 `YYYY/MM/DD`（生年月日入力は date ピッカー）。

## 9. 残課題・実装への申し送り（視覚）
- 必須/任意マークは現状の flat では非表示（要件は全氏名＋PII必須）。検証メッセージで補う方針＝実装時に必須の伝え方を最終確認（必要なら軽微な design 改訂）。
- 「LINE 認証済み」表示の最終配置（ヘッダ下 or リード位置）は実装時微調整。
- PII ブロックが長いので、スクロール量と送信ボタン位置に注意（A級＋全日協 ON が最長ケース）。
