## 正の所在

**リポジトリの `.claude/memory/` が唯一の正。** ローカルキャッシュ(`~/.claude/projects/.../memory/`)は作業用コピーに過ぎない。

**Why:** 家と会社の2環境で開発するため、git push/pullで同期できるリポジトリを正とする。手動で2箇所を管理すると同期忘れが起きる（2026-04-16に実際に発生）。

## セッション開始時

1. `git pull`
2. `.claude/memory/` → ローカルキャッシュにコピー（リポジトリが正なので上書き）

## セッション終了時

1. そのセッションで確定した**判断理由・運用ルール**をローカルキャッシュのmemoryファイルに追記
2. ローカルキャッシュ → `.claude/memory/` にコピー
3. worklog.md に作業記録を追記
4. git add → commit → push
5. 「コードから導出不可能な情報だけ書いたか？」を自問

## メモリに書くもの（コードから読み取れない情報のみ）

- 却下した選択肢とその理由
- 運用ルール・ビジネスロジックの前提（例: 「未回答=不参加扱い」）
- スコープ判断（例: 「団体戦は後日追加」）
- ユーザーのフィードバック・指摘

## メモリに書かないもの

- テーブル名・カラム名・型定義 → スキーマを読めばわかる
- APIエンドポイント → routes/を読めばわかる
- ファイルパス・ディレクトリ構成 → codebaseを読めばわかる
- 実装の詳細手順 → git diffを読めばわかる
