# Yahoo!メール大会案内自動処理に関する調査報告書

- 作成日: 2026-04-17
- 作成者: Codex
- 対象: Yahoo!メール受信内容を読み取り、大会案内メールのみアプリ登録し、非該当は通知してスルーする運用可否の調査

## 1. 調査目的

以下の実現可能性を確認する。

1. Yahoo!メールを自動で取得し内容を解析できるか
2. 大会案内メールを判定してアプリに登録できるか
3. 大会案内でないメールを通知してスルーできるか
4. 文面や添付形式がばらつく運用に耐えられるか

## 2. 調査範囲

- 実装は未着手（本書は調査報告）
- 実メールを使った接続・読取・添付解析の技術検証を実施
- 対象形式:
  - 本文: text/plain, text/html
  - 添付: PDF, XLSX
  - 将来想定: DOCX, DOC

## 3. 検証結果サマリー

### 3.1 接続方式

- 結論: `Webスクレイピング` ではなく `IMAP` が実用的
- 使用先:
  - `imap.mail.yahoo.co.jp:993 (SSL)`
- IMAP許可設定後に認証成功

### 3.2 受信一覧取得

- 2026-04-17 (JST) の Inbox 受信を抽出し、7件取得できることを確認
- 件名・送信者・日時の一覧化が可能

### 3.3 個別メール読取

- 件名 `初心者大会案内` のメールを特定し、以下を抽出可能
  - Date / From / To / Subject / Message-ID
  - 本文テキスト
  - 添付ファイル一覧

### 3.4 添付ファイル読取

- 対象メール添付2件を解析
  - `260530北海道初心者大会.pdf`
  - `260530北海道初心者大会参加申込書.xlsx`
- PDFから大会要項（日時・場所・参加費・申込締切・振込先等）を抽出できることを確認
- XLSXから申込書の項目（所属会、責任者、参加者欄、申込先等）を抽出できることを確認

## 4. 実現可否の結論

結論として、要件は実現可能。

1. 受信取得: 可能
2. 大会案内判定: 可能
3. アプリ登録連携: API仕様があれば可能
4. 非該当通知: 通知先仕様があれば可能
5. 再処理防止: Message-ID管理で可能

## 5. AI利用要否

### 5.1 結論

- 文面が一定ならルールベースのみでも運用可能
- 文面がバラバラという前提では、AI利用を推奨

### 5.2 推奨方式（ハイブリッド）

1. 前処理（非AI）
   - 新着取得、重複排除、本文/添付テキスト化
2. 判定・抽出（AI）
   - 大会案内判定
   - 必須項目抽出（大会名、日時、場所、費用、締切など）
3. 後処理（非AI）
   - 高信頼: 自動登録
   - 中信頼: 要確認通知
   - 低信頼: スルー通知

## 6. 想定システム構成（実装案）

1. Mail Collector
   - IMAPで未処理メール取得
2. Content Extractor
   - 本文抽出、PDF/XLSX/DOCX/DOC解析
3. Classifier
   - ルール + AIで判定
4. App Connector
   - 大会案内データをアプリAPIへ送信
5. Notifier
   - Slack/Discord/LINE/メール等へ結果通知
6. Audit Store
   - Message-ID、判定理由、登録結果、エラー履歴を保存

## 8. 想定リスクと対策

### 8.1 認証・セキュリティ

- リスク:
  - 資格情報漏えい
  - 認証方式変更で接続不能
- 対策:
  - 通常パスワードは使わず App Password または OAuth2 を利用
  - 認証情報は環境変数/シークレット管理
  - ログは個人情報をマスク

### 8.2 判定誤り（誤検知/見逃し）

- リスク:
  - 非大会メールを登録
  - 大会案内を見逃し
- 対策:
  - 信頼度閾値で自動/要確認を分岐
  - 判定理由を保存し改善ループを回す
  - 週次で誤判定レビュー

### 8.3 添付形式の多様化

- リスク:
  - DOCやスキャンPDFで抽出不能
- 対策:
  - DOCX標準対応
  - DOCは変換ツール経由
  - 画像PDFはOCR対応をオプション追加
